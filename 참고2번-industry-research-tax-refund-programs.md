## 1. 경정청구 제도 개요

### 1.1 법적 근거

| 항목 | 내용 |
|------|------|
| 근거법령 | 국세기본법 제45조의2 |
| 정의 | 세금을 과다 납부한 경우 관할 세무서장에게 환급을 요청하는 제도 |
| 신청 기한 | 법정 신고기한 경과 후 **5년 이내** |
| 처리 기간 | 접수일로부터 **2개월 이내** (기한 후 신고는 3개월) |
| 환급 기간 | 통상 1.5~2개월 내 검토 및 환급 완료 |
| 처리 관서 | 20~30억 미만: 세무서, 이상: 지방청 |

### 1.2 필요 서류 (공통)

- 경정청구서 (국세기본법 시행규칙 별지 제16호의2서식)
- 당초 제출분 관련 서류 (지급명세서, 소득공제 신고서)
- 추가 공제 관련 증빙서류
- 사유별 증빙 (의료비, 교육비, 기부금, 임대차계약서, 세금계산서 등)

### 1.3 경정청구 사유 유형

| 유형 | 설명 | 사례 |
|------|------|------|
| 일반 경정청구 | 세법상 공제/감면 누락 | 통합고용세액공제 미적용 |
| 후발적 경정청구 | 판결/결정 등 후발적 사유 | 세법 개정 소급 적용 |
| 기한 후 신고 | 무신고 후 자진신고 | 종합소득세 미신고분 |

## 2. 국내 주요 서비스 분석

### 2.1 택스봇R (TaxBotR) — 인포텍코퍼레이션

| 항목 | 내용 |
|------|------|
| 유형 | B2B (세무사무소 대상 솔루션) |
| 기술 | RPA(Robotic Process Automation) + AI 알고리즘 |
| 대상 | 법인사업자 + 개인사업자 |
| 도입 규모 | 1,000개 이상 세무사무소 |

**업무 처리 프로세스**:
```
1. 자동 데이터 수집 (3분 이내)
   - 홈택스 공인인증서 로그인
   - 최근 6년치 종합소득세/법인세 신고 자료 수집
   - 고용산재 토털사이트 데이터 수집

2. AI 알고리즘 분석
   - 과납 세금 환급대상자 확인
   - 환급금 자동 계산

3. 세액공제 진단
   - 누락된 세액공제 항목 자동 진단
   - 사업자 핵심 공제/감면 항목 집중 점검 (통합고용/투자/창업/중소특별/R&D/사보 등)
   - ※ "200여 가지"는 비즈넵의 수치이며, 택스봇R은 B2B 세무사무소용으로 사업자 핵심 항목에 특화

4. 세무사 검토
   - 전문 세무사의 최종 검토

5. 경정청구서 작성 및 제출
```

**파트너십**: 이지샵, 세무사랑Pro 2.0, 비즈넵, 캐시노트, 팜택스 등

**RPA 자동수집 데이터 상세** (v1.0 보완):

| 수집 소스 | 카테고리 | 세부 수집 항목 |
|----------|---------|-------------|
| **홈택스** | 과거 신고서 | 종합소득세/법인세 신고서(6년치), 세액조정계산서, 공제감면세액계산서, 최저한세 조정계산서 |
| | 세금계산서 | 전자세금계산서 매출/매입 합계표, 공급가액/세액, 거래처별 상세(6년치) |
| | 원천징수 | 원천징수이행상황신고서, 지급명세서(근로/사업/기타), 간이지급명세서 |
| | 사업자 기본 | 사업자등록 상세(업종코드, 개업일, 대표자), 사업장 현황신고서 |
| | 부가세 | 부가가치세 신고서(6년치), 매출/매입 세금계산서 합계 |
| | 기타 | 현금영수증 매출/매입, 고정자산 감가상각 명세서, 환급 내역 |
| **고용산재토털** | 고용보험 | 월별 피보험자 수, 취득/상실일, 연령, 성별, 고용형태(정규/비정규/일용) |
| | 산재보험 | 근로자 수, 보수총액 |
| | 보험료 | 고용보험/산재보험 사업주 부담분 납부 상세 |
| | 근로자 유형 | 청년(15~34세)/장애인/60세↑/경력단절여성 구분 |

**6년치 수집 사유별 상세**:

| 연차 | 수집 사유 | 관련 공제/감면 |
|------|----------|--------------|
| 당해~5년 전 | 경정청구 대상 기간 (국기법 §45의2) | 전체 |
| 6년 전 (+1년) | 직전연도 비교 기준 (상시근로자 수 증감, R&D비 증가분 산정) | 통합고용(§29의8), R&D(§10) |
| 5개년 연속 | R&D 세액공제 증가분 방식 직전 4년 평균 | R&D(§10) |
| 3개년 연속 | 통합투자세액공제 추가공제 직전 3년 평균 초과분 | 통합투자(§24) |

### 2.2 삼쩜삼 (3.3) — 자비스앤빌런즈

| 항목 | 내용 |
|------|------|
| 유형 | B2C (프리랜서, N잡러, 인적용역소득자) |
| 기술 | AI LLM + 개인화 엔진 |
| 실적 | 누적 환급액 1조 6,700억원, 1,220만 건 신고 데이터 |
| 수수료 | 환급액의 10~20% |

**업무 처리 프로세스**:
```
1. 간편인증 로그인 (최대 1분)
   - 최근 5년간 종합소득세 자료 자동 수집

2. 세무사 무료 검토
   - 감면 받은 공제 + 누락 공제 내역 확인

3. 체크리스트 작성
   - 사용자 추가 정보 입력

4. 예상 환급액 확인 (영업일 1~3일)
   - 최대 4,608개 맞춤 화면(플로) 제공

5. 서비스 이용료 결제 → 경정청구 제출 → 환급 (약 2개월)
```

**차별화**: 세무 LLM 기반 30초 서류 검토, 추징 시 최대 50만원 보상

### 2.3 비즈넵 환급 — 지엔터프라이즈

| 항목 | 내용 |
|------|------|
| 유형 | B2C (개인/법인 사업자) |
| 실적 | 회원 100만명, 누적 환급액 3,200억원 |
| 수수료 | 환급액의 30% (책임환불제) |
| 평균 환급금 | 960만원 |

**업무 처리 프로세스**:
```
1. 무료 조회 (30초)
   - 200여 가지 세액공제/감면 항목 AI 조회

2. 신청 버튼 클릭

3. 세무사 더블체크 검토

4. 경정청구 제출 → 환급 (약 2개월)
```

### 2.6 더낸세금 — 세무법인 혜움 [v1.0 보완]

| 항목 | 내용 |
|------|------|
| 유형 | B2B/B2C (기존 기장 고객 + 신규 고객) |
| 기업고객 | 1.7만개 기업 기장 데이터 보유 |
| 차별화 | 기장 과정에서 이미 작성한 세무조정계산서 원본 보유 → 재검토 정밀도 높음 |

**데이터 수집 방식 (기장고객 vs 신규고객 이원화)**:

| 구분 | 데이터 소스 | 수집 방식 | 범위 |
|------|-----------|----------|------|
| **기장 고객** | 세무법인 자체 ERP | 직접 접근 | 재무제표, 세무조정계산서, 원천징수이행상황, 지급명세서 (수년치) |
| | 홈택스 | 위임장 기반 직접 조회 | 과거 신고 내역, 세금계산서, 현금영수증 |
| | 4대보험 | EDI/토털사이트 직접 조회 | 피보험자 이력, 보험료 납부 |
| **신규 고객** | 홈택스 | 간편인증 기반 RPA 수집 | 택스봇R과 유사 범위 |

**법인세 경정청구 특화 입력항목**:
- 세무조정 오류 재검토: 기장 시 작성한 세무조정계산서 원본 재점검 (손금불산입 과다, 익금불산입 누락)
- 이월결손금 최적 활용: 연도별 결손금 잔액 + 공제 순서 최적화
- 중소기업 판단 재검토: 매출액/자산총액/독립성 기준 재확인
- 창업 요건 재점검: 법인전환, 동종사업 승계 여부 세부 검증

| 항목 | 내용 |
|------|------|
| 출시 | 2025년 3월 홈택스 내 정식 출시 |

| 대상 | 전체 납세자 (개인 중심) |

**자동 환급 산출에 활용하는 입력 데이터**:

|------------|---------|:--------:|
| 소득 정보 | 원천징수영수증, 지급명세서, 사업소득 수입금액 | O |

## 6. 환급금 계산 로직

### 6.1 전체 계산 흐름 (8단계)

```
┌─────────────────────────────────────────────────────────────┐
│  STEP 1: 과세표준 재산정                                      │
│  ├─ [INC] 총수입금액 - 필요경비 = 소득금액                      │
│  │        소득금액 - 소득공제 = 과세표준                         │
│  └─ [CORP] 당기순이익 ± 세무조정 - 이월결손금 = 과세표준          │
│                          ↓                                    │
│  STEP 2: 산출세액 계산                                        │
│  ├─ [INC] 과세표준 × 종합소득세율 (6~45%, 8단계 누진)           │
│  └─ [CORP] 과세표준 × 법인세율 (9~24%, 4단계 누진)              │
│                          ↓                                    │
│  STEP 3: 적용 가능한 감면/공제 식별                             │
│  ├─ 업종, 지역, 규모, 기간 등 요건 충족 여부 확인                │
│  └─ 적용순서: 감면 → 이월불가 공제 → 이월가능 공제               │
│                          ↓                                    │
│  STEP 4: 최적 조합 선택 (상호배제 고려)                         │
│  ├─ 조특법 §127④ 상호배제 규칙 적용                             │
│  ├─ Branch & Bound / Greedy 알고리즘                           │
│  └─ 환급액 최대화 조합 선택                                     │
│                          ↓                                    │
│  STEP 5: 최저한세 적용                                        │
│  ├─ [INC] 산출세액 × 35%/45%                                  │
│  └─ [CORP] 과세표준 × 7%/10%/12%/17%                          │
│                          ↓                                    │
│  STEP 6: 농어촌특별세 계산                                     │
│  ├─ 감면세액 × 20% (과세 항목)                                 │
│  └─ 비과세 항목: §6, §7, §10                                  │
│                          ↓                                    │
│  STEP 7: 환급세액 산출                                        │
│  └─ 환급액 = 기존 결정세액 - 경정 후 결정세액                    │
│                          ↓                                    │
│  STEP 8: 환급가산금 + 지방소득세 + 실질환급액                    │
│  ├─ 환급가산금 = 환급세액 × 이자율 × 경과일수/365               │
│  ├─ 지방소득세 환급 = 본세 환급 × 10%                           │
│  └─ 실질환급액 = 환급세액 + 환급가산금 + 지방소득세 - 농특세      │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 최적 조합 알고리즘 (의사코드)

```
function findOptimalCombination(credits, exclusionRules):
    if credits.count <= 15:  // Branch & Bound
        bestCombo = branchAndBound(credits, exclusionRules)
    else:  // Greedy
        bestCombo = greedy(credits, exclusionRules)

    for each combo in validCombinations:
        netRefund = computeRefund(combo)  // 감면/공제 적용
        netRefund -= computeAMT(combo)     // 최저한세 차감
        netRefund -= computeAgriTax(combo) // 농특세 차감
        if netRefund > bestRefund:
            bestRefund = netRefund
            bestCombo = combo

    return bestCombo
```

## 7. 주요 감면/공제 계산 공식

### 7.1 통합고용세액공제 (조특법 §29의8)

```
공제액 = 증가 인원 × 1인당 공제액 × 공제기간(3년 또는 2년)

증가 인원 = 당해 상시근로자 - 직전 상시근로자
상시근로자 수 = Σ(매월 말 인원) / 12개월

[중소기업 1인당 공제액]
┌──────────────────────┬──────────┬──────────┐
│ 유형                  │ 수도권    │ 비수도권  │
├──────────────────────┼──────────┼──────────┤
│ 청년/장애인/60세↑/경단녀 │ 1,450만원 │ 1,550만원 │
│ 일반 근로자            │ 850만원   │ 950만원   │
└──────────────────────┴──────────┴──────────┘

주의: 공제 후 2년 내 상시근로자 감소 시 추징
```

### 7.2 통합투자세액공제 (조특법 §24)

```
공제액 = 투자금액 × 기본공제율 + 초과투자분 × 추가공제율

[기본공제율]
- 중소기업: 10% (신성장기술 12%)
- 중견기업: 5% (신성장기술 7%)
- 대기업: 1~2%

[추가공제율]
- 직전 3년 평균 초과분 × 3~4%

대상: 기계장치, 설비, 소프트웨어, 특허권 등
제외: 중고자산, 승용차, 수도권 과밀억제권역 일부
주의: 투자 후 2년 내 처분/임대 시 추징+이자
```

### 7.3 창업중소기업 세액감면 (조특법 §6)

```
감면세액 = 해당 사업 소득 × 감면율

[감면율]
┌──────────────────┬──────────┬──────────┐
│ 구분              │ 수도권 외 │ 수도권    │
├──────────────────┼──────────┼──────────┤
│ 청년 창업 (15~34세)│ 100%     │ 50%      │
│ 일반 창업          │ 50%      │ 해당없음  │
│ 인구감소지역       │ 100%     │ -        │
└──────────────────┴──────────┴──────────┘

감면기간: 최초 소득 발생연도 + 4년 (총 5년)
요건: 중소기업, 제조업 등 지정업종, 창업 5년 이내
```

### 7.4 중소기업 특별세액감면 (조특법 §7)

```
감면세액 = 해당 사업장 소득 × 감면율

[감면율 (업종/지역/규모별)]
- 제조업, 광업: 수도권 외 30% / 수도권 20%
- 도소매업: 수도권 외 15% / 수도권 10%
- 기타 서비스: 5~15%

한도: 연 1억원
      상시근로자 감소 시: 1억원 - (감소 인원 × 500만원)

적용기간: 2028년 12월 31일까지
제외: 소비성서비스업, 부동산업 등
```

### 7.5 R&D 세액공제 (조특법 §10)

```
[일반 R&D]
공제액 = 연구개발비 × 공제율

- 중소기업: 25~30%
- 중견기업: 15~20%

[신성장/원천기술]
- 중소기업: 30~40%
- 중견기업: 20~30%

[국가전략기술]
- 중소기업: 40~50%
- 중견기업: 30~40%

대상비용: 인건비, 재료비, 위탁R&D비, 감가상각비
R&D 3단계 최저한세 배제: 국가전략 > 신성장 > 일반 순
```

### 7.6 사회보험료 세액공제 (조특법 §30의4)

```
공제액 = Σ(사업주부담분 × 공제율, 근로자별)

[공제율]
- 청년(15~29세), 경력단절여성: 100%
- 일반: 50%

대상: 국민연금, 건강보험, 고용보험, 산재보험 사업주 부담분
요건: 중소기업, 상시근로자 증가분에 대해 적용
기간: 2년간
```

### 7.7 기장세액공제 (개인사업자 전용)

```
공제액 = 산출세액 × (사업소득금액 / 종합소득금액) × 20%
한도: 100만원

요건: 간편장부대상자가 복식부기로 기장
중복배제 비대상 (조특법 공제와 별도 적용 가능)
```

### 7.8 최저한세 계산

```
[법인세 - 과세표준 기준]
- 중소기업: 7%
- 중견기업 1,000억 이하: 10%
- 중견기업 1,000억 초과: 12%
- 대기업: 17%

[종합소득세 - 산출세액 기준]
- 산출세액 3천만원 이하: 35%
- 산출세액 3천만원 초과: 45%

IF (감면/공제 후 세액 < 최저한세):
    결정세액 = 최저한세  // 초과분은 이월공제
ELSE:
    결정세액 = 감면/공제 후 세액
```

### 7.9 환급가산금 계산

```
환급가산금 = 환급세액 × 이자율 × (경과일수 / 365)

이자율: 연 3.1% (2025년 기준, 시중금리 연동 변동)
기산일:
  - 본세: 법정신고기한 다음날 ~ 환급결정일
  - 중간예납: 중간예납 납부일 다음날 ~ 환급결정일
  - [INC] 기산일: 5.31(일반) 또는 6.30(성실신고확인대상)

예시: 환급세액 1,000만원, 180일 경과
     = 10,000,000 × 0.031 × (180/365) = 152,877원

주의: 환급가산금 1원 미만 절사
      환급가산금은 익금불산입 (법인세법 §18 제4호)
```

## 10. 자동화 기술 트렌드

### 10.1 기술별 활용 현황

| 기술 | 활용 분야 | 대표 사례 | 효과 |
|------|---------|----------|------|
| **RPA** | 홈택스 자동수집, 서식작성 | 택스봇R (3분 6년치 수집) | 반복업무 90%↓ |
| **AI/ML** | 공제 적격성 판단, 패턴분석 | 삼쩜삼 LLM (4,608 맞춤 플로) | 서류검토 30초 |
| **OCR** | 증빙서류 자동인식 | CLOVA OCR, 딥러닝 OCR | 오류 감소, 효율 향상 |
| **NLP** | 세법/예규/판례 자동해석 | 국세청 AI 세무상담 (2027~) | 세법 변경 자동 반영 |
| **빅데이터** | 환급 패턴 발견, 탈세 탐지 | 국세청 AI 조사 (적중률 80%) | 누락 항목 자동 발견 |

### 10.2 AI 도입 로드맵

| 시기 | 기술 | 내용 |
|------|------|------|
| 현재 (2025~2026) | RPA + Rule-based AI | 데이터 수집 자동화, 규칙 기반 공제 판단 |
| 단기 (2026~2027) | LLM + OCR 고도화 | 세법 자동 해석, 증빙서류 자동 분류, 국세청 AI 시스템 연동 |
| 중기 (2027~2029) | 예측 AI + 블록체인 | 실시간 환급 예측, 증빙 관리 블록체인, 세법 개정 즉시 반영 |
| 장기 (2030~) | Zero-Touch Filing | 완전 자율 세무 신고, 글로벌 환급 서비스 |

### 10.3 국세청 AI 전환 동향

- **2025년**: 세무조사 대상 선정 40년 만에 전산→AI 전환 (적중률 70~80%)
- **2027년**: 홈택스 등 납세시스템 AI 기반 전면 개편
- **AI 세무상담**: 세법, 예규, 판례 학습 기반 정확한 상담서비스
- **빅데이터 분석**: 신고서 항목 '자동(모두)채움' 대폭 확대
- **민간 견제**: 국세청 원클릭 환급 서비스 (수수료 0원)

1. **홈택스 API 연동 설계** — 데이터 수집 자동화는 업계 핵심 경쟁력. M1 모듈에 홈택스 API 어댑터 설계 추가
2. **세무사 검토 워크플로** — 업계 전체가 "AI 1차 + 세무사 2차" 모델 채택. 검토/승인/반려 워크플로 추가
3. **사후관리 알림** — 고용유지(2년), 자산처분(2년), 사업유지(3년) 모니터링 기능을 M7에서 Phase 1으로 상향

**단기 반영 (Phase 2)**

4. **OCR 증빙 자동인식** — 세금계산서, 임대차계약서, 의료비영수증 등 OCR 처리
5. **업종별 맞춤 시나리오** — 제조업/서비스업/건설업 등 업종별 최적화된 점검 플로
6. **고객 리포트 고도화** — PDF 자동생성, 모바일 최적화, 근거법령 자동 첨부

**중기 반영 (Phase 3)**

7. **AI 패턴 학습** — 누적 데이터 기반 누락 항목 자동 탐지 ML 모델
8. **경정청구서 자동 작성** — 별지 제16호의2서식 자동 생성 및 홈택스 자동 제출
9. **실시간 심사 추적** — 국세청 처리 상황 자동 모니터링
